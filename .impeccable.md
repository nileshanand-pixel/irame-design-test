## Design Context

### Users
Finance and audit professionals at enterprise organizations — both internal teams (controllers, audit leads) and external clients. They work in high-stakes, accuracy-critical environments and need to trust the tool implicitly. They're power users who spend long sessions in the product.

### Brand Personality
AI, modern, intelligent, accurate.
Voice: confident and precise — never playful, never vague. Every UI element should feel purposeful.

### Aesthetic Direction
Reference: StackAI (https://www.stackai.com/) — enterprise-grade SaaS, clean and dense, premium positioning with generous whitespace in key areas. Clean card-based layouts, modular information blocks, trust signals.

Brand color: Purple `#6A12CD` / `rgba(106, 18, 205, 1)` — assertive, intelligent.
Deep navy-purple: `rgba(38, 6, 74, 1)` — used for primary text at opacity steps.
Light mode only. No dark mode in production.
Font: Inter — appropriate for the enterprise SaaS context; keep it consistent.

Anti-references: Avoid consumer-app friendliness (Notion-style rounded playfulness), avoid glowing neon dark mode aesthetics, avoid oversized decorative elements.

### Design Principles
1. **Density with breathing room** — pack information efficiently but never crowd. Use whitespace to separate sections, tighten within them.
2. **Type hierarchy is everything** — in data-dense UIs, size + weight + color must create an unmistakable scan path. Names > metadata, labels > body.
3. **Proportionality** — every element's size should be proportional to its importance. UI chrome (icons, labels, metadata) stays small; user content/actions stay prominent.
4. **Purple with restraint** — use the brand purple for active states, CTAs, and focus — not decoration. It should signal "this matters."
5. **Precision over personality** — this is a tool for audit professionals. Clarity and accuracy of information display trumps visual flourish.
